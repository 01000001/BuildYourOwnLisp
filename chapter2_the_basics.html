Chapter 2 - The Basics

Setting up your environment

Before getting started programming C you'll have to install a C compiler and a text editor. The C compiler is what transforms the C source code into a program your computer can understand. A text editor a program that allows you to edit text files suitable for programming. The installation process for either of these is different depending on what operating system you are running. For this, and the rest of the tutorial, I am going to assume you are familiar with using the console. If you are not familiar you can use this crash course to get up to speed.

Linux

On Linux systems to instally a C compiler you will have to install some packages. If you are running Ubuntu or Debian you can install everything you need with the following command.

sudo apt-get install build-essential

If you are running Fedora or a similar Linux variant you can use this command.

su -c "yum groupinstall development-tools"

For a text editor I recommend using gedit or whatever other basic text editor comes installed with your distribution

Mac

If you are on Mac OSX then you must install the latest version of XCode from Apply. This could take a long time as typically XCode is a big peice of software to install. If you face troubles search online for "installing xcode" and follow any advice shown.

Once XCode is installed you should have access to a C compiler from the console and be ready to go.

A simple text editor that can be used on Mac is TextWrangler. Please don't use XCode for text editing as using an IDE hinders greatly the learning of a new language.

Windows

On windows I recommend downloading and installing MinGW. This will install a C compiler on your computer and allow you to run it from the console as if you were on a Linux system.

My text editor of choice on windows is Notepad++, which is an excellent editor in all respects. If you have another preference please go with it, though don't use Visual Studio as it does not have proper support for C programming.

Testing Your Compiler

To test if your C compiler is installed correctly type the following into the console.

$ cc --version
  cc (rubenvb-4.7.2-release) 4.7.2
  ...

If you get something like the following echoed back your C compiler is installed correctly and you are ready to go!

Hello World

Now that your environment is set up, start by opening your text editor and inputting the following code. Save this file somewhere as hello_world.c

#include <stdio.h>

int main(int argc, char** argv) {
  puts("Hello, world!");
  return 0;
}

There are three parts to this program. First we include a header called stdio.h. This allows us to use standard input and output functions which come included with C. One of these is the puts function you see later on in the program. This function outputs a string to the console.

Secondly we create a function called main, returning an int, and taking as input an int called argc and a char** called argv. All C programs must contain this function. It acts as the entry point to execution. When you run your program it always starts at main and continues from there. For now don't worry about the inputs argc and argv. We will cover those later.

Inside main we have puts("Hello, world!");. This outputs Hello, world! to the console. The function puts stands for put string. Also is the line return 0;, which tells the main function to finish and return 0. The 0 indicates there have been no errors running the program.

Save this code somewhere and browse to it on the command line. Next we want to compile the code and output an actual program we can run on our computer. For this we want to open up a console and browse to the directly we have saved our file. We can then run the following command.

cc -std=c99 -Wall hello_world.c -o hello_world

This compiles the code in hello_world.c, reporting any warnings, and outputs the program to a new file hello_world. If successful you should see a new file called hello_world in the current directory. This can be called by typing ./hello_world. If all goes to plan you should see a friendly Hello, world! message appear.

Debugging

When there is a problem in a C program, it won't throw an error message telling you what when wrong. It will simply crash and close. This makes debugging C programs harder than usual. It can be an art in itself, far beyond the scope of this tutorial.

My first port of call for debugging is to print out lots of information as the program is running. Using this method I can try to isolate exactly what part of the code is incorrect and what, if anything, is going wrong up until the crash. For beginners I would recommend this technique. It is active debugging - which is the important thing. As long as you are doing something, the process is less painful and the temptation to give up, or smash your stupid computer, is lessened.

For people feeling more confident a program called gdb can be used to debug your C programs. This can be difficult and complicated to use, but it is also very powerful and can give you extremely valuable information and what went wrong and where.

For those using Linux valgrind can be used to aid the debugging of memory leaks and other more nasty errors. Valgrind is a tool that can save you hours to days of debugging. It does not take much to get proficient at it, so investigating it is highly recommended.
