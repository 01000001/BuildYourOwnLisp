Compiler & Text Editor

Before getting started programming C you'll have to install a compiler and a text editor.

The compiler is what transforms the C source code into a program your computer can run. A text editor will allow you to edit text files in a way suitable for programming.

The installation process for these is different depending on what operating system you are running.

Compiling and running C programs is also going to make use of the command line. For this, and the rest of the book, I am going to assume you are familiar with using the command line. If you are not familiar then search online for information on using it, relevant to your operating system.

Linux

On Linux systems to install a C compiler you will have to download some packages. If you are running Ubuntu or Debian you can install everything you need with the following command.

sudo apt-get install build-essential

If you are running Fedora or a similar Linux variant you can use this command.

su -c "yum groupinstall development-tools"

For a text editor I recommend using gedit or whatever other basic text editor comes installed with your distribution. If you are a Vim or Emacs user these are also fine. Please don't use an IDE. It isn't required for such a small project and wont help in understanding what is going on.

Mac

If you are on Mac OSX then you should install the latest version of XCode from Apple. This could take a long time as typically XCode is a big peice of software to install. If you are unsure of how to do this you can search online for "installing xcode" and follow any advice shown. You may also have to install the XCode command line tools in a separate package.

Once XCode is installed you should have access to a C compiler from the command line and be ready to go.

A simple text editor that can be used on Mac is TextWrangler. If you have a different preference this is fine. Please don't use XCode for text editing as this is a small project and using an IDE wont help you understand what is going on.

Windows

On windows I recommend downloading and installing MinGW. This will install a C compiler and allow you to run it from the command line cmd.exe. It will create programs ending in the .exe suffix which should be run by using their full name without the leading ./.

My text editor of choice on windows is Notepad++. If you have another preference this is fine. Please don't use Visual Studio as it does not have proper support for C programming. It you attempt to use it you will run into many problems.

Testing

To test if your C compiler is installed correctly type the following into the command line.

$ cc --version
  cc (rubenvb-4.7.2-release) 4.7.2
  ...

If you get something like the following echoed back, then your C compiler is installed correctly and you are ready to go!

Don't include the $ sign! This is simply used to indicate this command should be run from the command line.

Hello World

Now that your environment is set up, start by opening your text editor and inputting the following code. Create a directory where you are going to work on this book and save this file as hello_world.c. This is your first C program!

#include <stdio.h>

int main(int argc, char** argv) {
  puts("Hello, world!");
  return 0;
}

Depending on your programming experience this may look like a lot of crazy symbols that make very little sense. I'll try to explain it step by step.

In the first line we include what is called a header. This statement allows us to use the functions from stdio.h, the standard input and output library which comes included with C. One of these is the puts function you see later on in the program.

Next we declare a function called main. This function is declared to return an int, and take as input an int called argc and a char** called argv. All C programs must contain this function. It acts as the entry point to execution. When you run a C program it always starts at main and continues from there.

For now don't worry about the inputs argc and argv. We will cover those in later chapters.

Inside main we see the line puts("Hello, world!");. This is a call to the function puts, which outputs Hello, world! to the command line. The function puts stands for put string.

Finally we see return 0;. This tells the main function to finish and return 0. When a C program returns 0 this indicates there have been no errors running the program.

Compilation

Before we can run this program we need to compile it. This will produce the actual executable we can run on our computer. Open up a terminal and browse to the directory that hello_world.c is saved in. Compile you program using the following command.

$ cc -std=c99 -Wall hello_world.c -lm -o hello_world

This compiles the code in hello_world.c, reporting any warnings -Wall, and outputs -o the program to a new file hello_world.

If successful you should see the output file in the current directory. This can be run by typing ./hello_world (or hello_world.exe on windows). If all goes to plan you should see a friendly Hello, world! message appear as output.

Congratulations! You've just compiled and run your first C program.

Compiler Errors

If there are some problems with your C program the compilation process may fail. These issues can range from simple syntax errors, to other complicated problems that are hard to understand.

Sometimes the error message from the compiler will make sense, but if you are having trouble understanding it try searching online for it. You should see if you can find a concise explaination of what it means, and work out how to correct it. Remember this: there are many people before you who have struggled with exactly the same problems.

Sometimes there will be many compiler errors stemming from one source. Always work through compiler errors from first to last.

Runtime Errors

Sometimes the compiler will compile a program, but when you run it it will crash. Debugging C programs in this situation is hard. It can be an art far beyond the scope of this book. But it still needs to be done.

Rage • A poor debugging technique

My first port of call for debugging a crashing C program is to print out lots of information as the program is running. Using this method I can try to isolate exactly what part of the code is incorrect and what, if anything, is going wrong up until the crash. For beginners I would recommend this technique. It is active debugging - which is the important thing. As long as you are doing something, the process is less painful and the temptation to give up is lessened.

For people feeling more confident a program called gdb can be used to debug your C programs. This can be difficult and complicated to use, but it is also very powerful and can give you extremely valuable information and what went wrong and where. Information on how to use gdb can be found online.

For those using Linux valgrind can be used to aid the debugging of memory leaks and other more nasty errors. Valgrind is a tool that can save you hours, or even days, of debugging. It does not take much to get proficient at it, so investigating it is highly recommended. Again information on how to use it can be found online.

Reference Documentation

If you spot a C function in some example code that you don't recognize you might wonder what it does. In this case you will want to look toward the reference documentation, which explains all the functions included in the C standard library.
